# Data Dictionary

Detailed description of columns, data types, nullability, and definitions for all serving layer tables.

---

## 1. Table Definitions

### `serving.fact_sales_daily`
| Column Name | Data Type | Nullable | Primary Key | Description |
|---|---|---|---|---|
| `sale_date` | DATE | NO | YES | Aggregated calendar date for order transactions |
| `total_revenue` | DECIMAL(12,2) | NO | NO | Total price + freight value of daily transactions |
| `total_orders` | INT | NO | NO | Total quantity of orders purchased daily |
| `_loaded_at` | TIMESTAMPTZ | NO | NO | Metadata audit column indicating row ingestion time |

### `serving.dim_customers_clv`
| Column Name | Data Type | Nullable | Primary Key | Description |
|---|---|---|---|---|
| `customer_unique_id` | VARCHAR(50) | NO | YES | Anonymized unique customer identifier |
| `total_orders` | INT | NO | NO | Total count of orders placed by customer |
| `lifetime_value` | DECIMAL(12,2) | NO | NO | Total cumulative monetary value spent by customer |
| `avg_order_spend` | DECIMAL(12,2) | NO | NO | Average order basket value |
| `is_repeat_customer` | BOOLEAN | NO | NO | Flag indicating if user has bought more than once |
| `customer_segment` | VARCHAR(20) | NO | NO | Segmentation category: VIP, High Value, Standard |
| `_loaded_at` | TIMESTAMPTZ | NO | NO | Metadata audit column indicating row ingestion time |
